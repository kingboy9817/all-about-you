# Disclosure

Use this to define when names, employers, referrers, and sensitive work details can appear in external materials.

## Reference policy

- Mention a reference by name only after the user approves that context.
- Use "former manager" or "operations leader" when a warm introduction is not active.
- Do not imply a reference has endorsed a specific application unless they have.

## Employer boundaries

- Public company names are allowed when already present in the KB and not marked sensitive.
- Private internal project details should be generalized unless cleared.
- Customer names should be anonymized by default.

## Safe anonymization examples

- "A B2B software team" instead of a specific customer.
- "Internal intake workflow" instead of a private tool name.
- "Regional operations team" instead of a small identifiable group.
